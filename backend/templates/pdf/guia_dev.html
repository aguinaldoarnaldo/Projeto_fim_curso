Guia Técnico de Continuidade do Projecto

1. Introdução

Este documento serve como manual de integração para desenvolvedores que se juntarão ao projecto de Gestão Escolar e Emissão de Documentos (IP Maiombe). O sistema foi construído com foco em escalabilidade, segurança e estética profissional.

2. Arquitetura do Sistema

  • Core: Django 5.2.4 (O coração do sistema).
  • Interface Admin: Django Unfold (Layout moderno e Dark Mode).
  • API Layer: Django REST Framework (DRF) para integração com frontend.
  • Segurança: SimpleJWT + Custom Authentication (SchoolJWTAuthentication).
  • Documentos: xhtml2pdf (Conversão de HTML para documentos oficiais).

3. Estrutura de Pastas (Backend)

  • core/: Configurações globais, settings.py e roteamento principal.
  • apis/models/: Modelagem modular (31 tabelas divididas por contexto).
  • apis/views/: Lógica de ViewSets e endpoints da API.
  • apis/services/: Camada Crítica - Centraliza as regras de negócio (PDF, Académico, Documentos).
  • apis/permissions/: Controle de acesso granular por Cargo (Diretor, Professor, Aluno).

4. Como Iniciar (Setup)

# 1. Instalar dependências
pip install -r requirements.txt

# 2. Migrar o banco de dados (PostgreSQL)
python manage.py migrate

# 3. Criar Superusuário (Acesso Admin)
python manage.py createsuperuser

# 4. Iniciar Servidor
python manage.py runserver

5. Fluxo de Geração de Documentos

O fluxo segue o padrão: Solicitação -> Pagamento -> Aprovação -> Geração. Quando um Diretor aprova uma solicitação via API ou Admin, o DocumentService aciona o PDFService, que renderiza o template HTML e salva o PDF no directório /media/documentos/.

6. Próximos Passos Recomendados

  1. Módulo Yasmin (IA): Implementar reconhecimento facial para chamadas automáticas.
  2. Notificações: Configurar envio de e-mails para alunos quando documentos forem aprovados.
  3. Integração de Pagamento: Conectar o RUPE ao gateway bancário real.
